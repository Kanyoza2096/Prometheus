You are building the frontend control layer for the Kanyoza Systems AI Platform — a production multi-tenant SaaS that automates social media content via AI. The backend is a Python Flask API at /api/v1/* with 60+ endpoints. The frontend is React + TypeScript + Tailwind CSS v4 + Framer Motion + Zustand + Recharts.

The backend endpoints ALL exist and return real data. The frontend currently only READS data (Dashboard, Monitoring, Health). It needs full CRUD control over every backend resource.

I need you to build the following pages. Each page must:
- Use the existing Zustand store at '../store/useStore' for restEndpoint, masterToken
- Use the existing API client at '../services/api' (import functions from there)
- Use Tailwind brand-* classes: brand-bg (#0A0E17), brand-surface (#141A2E), brand-border (#1E293B), brand-primary (#4F46E5), brand-text (#F1F5F9), brand-text-muted (#64748B), brand-elevated (#1C2541), brand-success (#10B981), brand-danger (#EF4444), brand-warning (#F59E0B), brand-accent (#06B6D4)
- Use shadow-glow-primary, shadow-glow-success, etc. from the theme
- Use font-mono for data, uppercase tracking-widest for labels
- Use Framer Motion for staggered fade-in animations (initial={{ opacity: 0, y: 10 }} animate={{ opacity: 1, y: 0 }})
- Show skeleton loading states while fetching
- Show empty states ("No posts yet", "No brands configured") when data is empty
- Show error states with retry button when API calls fail
- Be fully responsive (mobile bottom nav compatible)
- NEVER use mock data — all data from API calls
- Include search, filter, and pagination where applicable
